Основы перенаправления ввода-вывода в Linux
В данной практике вы научитесь работать с потоками ввода-вывода (STDIN, STDOUT и STDERR) и перенаправлять их в файлы, а также использовать конвейеры (pipes). Последовательно выполните все шаги и отправьте полный список использованных команд.

1) Подготовка рабочей среды
Создайте каталог redirection_lab и перейдите в него.
Создайте три файла со следующим содержимым (через команду echo или любым другим удобным способом):
words.txt — например, с тремя строками на ваш выбор, например:
Linux
Bash
Terminal

                  
numbers.txt — несколько чисел вразброс, по одному в строке (например, 3, 10, 1, 7 и т.п.).
errors.txt — пока пустой файл (просто создайте его).
2) Перенаправление STDOUT в файл
Выполните команду ls -l, перенаправив её стандартный вывод в файл ls_output.txt (используйте символ >).
Убедитесь, что на экране ничего не отобразилось, а содержимое появилось в файле ls_output.txt.
С помощью команды cat ls_output.txt проверьте, что в файл записался результат вывода команды ls -l.
3) Добавление (append) вывода в существующий файл
Ещё раз выполните команду date и добавьте (не перезаписывайте) её вывод в конец файла ls_output.txt, используя оператор >>.
Проверьте содержимое ls_output.txt, чтобы убедиться, что дата добавилась к уже существующим данным.
4) Перенаправление STDERR в отдельный файл
Выполните команду, которая заведомо вызовет ошибку, например ls /несуществующая_папка, и перенаправьте ошибки в error_log.txt (используйте 2>).
Убедитесь, что сообщение об ошибке появилось в error_log.txt и не вывелось на экран.
5) Перенаправление STDOUT и STDERR в один файл
Выполните команду, которая сгенерирует как успешный вывод, так и ошибку. Например:
ls words.txt /несуществующая_папка

                  
Перенаправьте результат (и STDOUT, и STDERR) в файл combined_output.txt, используя один из вариантов:
> combined_output.txt 2>&1
или &> combined_output.txt
Проверьте содержимое combined_output.txt, чтобы убедиться, что там есть и список файлов, и сообщение об ошибке.
6) Перенаправление ввода (STDIN)
Используя файл numbers.txt, выполните команду:
sort < numbers.txt

                  
Убедитесь, что числа выводятся в отсортированном порядке на экран.
Повторите операцию, перенаправив результат в новый файл sorted_numbers.txt:
sort < numbers.txt > sorted_numbers.txt
cat sorted_numbers.txt

                  
Проверьте, что файл sorted_numbers.txt содержит отсортированный список чисел.
7) Практика с конвейерами (pipes)
С помощью конвейера выведите содержимое файла words.txt, а затем построчно добавьте в конец каждой строки какой-нибудь текст (командой sed или awk) и при желании отсортируйте результат. Например:
cat words.txt | sed 's/$/ - Linux/' | sort

                  
Оцените, что произошло на каждом шаге цепочки.
Попробуйте сформировать другой конвейер, используя иные утилиты (например, grep, wc, uniq и т.д.), чтобы потренироваться передавать вывод одной команды на вход другой.
8) Итоговый отчёт (список команд)
Отправьте полный список команд, которые вы использовали при выполнении всех шагов задания (включая проверки содержания файлов). Пример начала отчёта:

mkdir redirection_lab
cd redirection_lab
echo "Linux" > words.txt
echo "3" > numbers.txt
...
(далее ваш полный список команд)

                  
На этом практика завершается. Убедитесь, что вы освоили:

Перенаправление STDOUT в файл (> и >>);
Перенаправление STDERR (2>);
Совместное перенаправление STDOUT и STDERR (&> или 2>&1);
Перенаправление STDIN (<);
Использование конвейеров (|).
Это важнейшие базовые навыки работы в терминале Linux, которые пригодятся вам в дальнейшей практике.
